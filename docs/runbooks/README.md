# Runbooks

Procedimentos operacionais e checklists para tarefas recorrentes. Documentam o **como fazer**, com passos concretos.

## Índice

- [`pr-checklist.md`](./pr-checklist.md) — o que verificar antes de abrir um PR.
- [`new-module.md`](./new-module.md) — como criar um novo módulo seguindo o estilo da casa (service-requests, proposals).

## Quando criar um runbook

Um procedimento merece runbook quando:

- É **recorrente** (acontece em mais de uma sessão).
- Tem passos **não-óbvios** que sem doc viram folclore.
- Erros frequentes têm **causa repetida** (ex.: esquecer migrate status).

Coisas pontuais (uma migração específica, um incidente único) ficam em entradas de "💬 Sessões Claude Code" no Notion, não aqui.
